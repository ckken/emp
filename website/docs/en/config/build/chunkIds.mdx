# build.chunkIds
:::tip
`named` Uses meaningful names as module ids for easier debugging. This option is enabled by default in development mode.
`deterministic` Uses hashed module identifiers as module ids, beneficial for long-term caching. This option is enabled by default in production mode.
:::
+ Type: `named` | `deterministic`

```ts title="emp-config.js"
import {defineConfig} from '@empjs/cli'
export default defineConfig(store => {
  return {
    build:{
        chunkIds:store.isDev?'named':'deterministic'
    }
  }
})
```
